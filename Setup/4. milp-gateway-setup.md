# MILP Gateway Setup Guide
> **Role:** Intelligent task offloading scheduler — uses Mixed Integer Linear Programming (MILP) to decide per-request whether to route to an Edge Node, Cloud Backup, or process locally, based on real-time utilization, task FLOPS, deadline, and UE bandwidth.

---

## Architecture Overview

```
UE Request (image)
       |
       ▼  POST /milp_predict
┌──────────────────────────────────────────────┐
│  MILP Gateway  (port 6001)                   │
│  milp_gateway.py                             │
│                                              │
│  1. Fetch UE bandwidth  ← throughput_server  │
│     (ogstun interface)    port 7000          │
│  2. Fetch edge /metrics (real-time util)     │
│  3. Run MILP solver (milp_core.py)           │
│  4. Route decision ──────────────────────┐   │
└──────────────────────────────────────────┼───┘
                    ┌─────────────────────┐|
                    │  Decision Tree      ││
                    │  Edge:edgeX ──────► Edge Nodes (30081–30090)
                    │  Cloud:cloud1 ───► Cloud Backup (172.16.23.128:8000)
                    │  Local ─────────► Return logical response
                    └─────────────────────┘
```

### Key Parameters

| Parameter | Value | Description |
|-----------|-------|-------------|
| Gateway Port | `6001` | MILP gateway listens here |
| Throughput Server Port | `7000` | Reads `ogstun` interface live BW |
| Edge Nodes | `30081–30090` | 9 edge inference pods |
| Cloud Fallback | `172.16.23.128:8000` | Docker backup inference server |
| MILP Solver | `HiGHS` (primary) / `CBC` (fallback) | Pyomo backend |
| Edge CPU Speed | `2.5 GHz` | Assumed per edge pod |
| Cloud CPU Speed | `10 GHz` | Assumed for cloud node |
| Deadline Slack | `+3s` | Relaxed deadline tolerance in MILP |

## Part 5 — Start the Throughput Server

The throughput server reads the `ogstun` tunnel interface (Open5GS UE subnet) every second and exposes live UE bandwidth at `/latest`. The MILP gateway calls this before every scheduling decision.

```bash
cd ~/Desktop/MILP
uvicorn throughput_server:app --host 0.0.0.0 --port 7000
```

Expected output:
```
INFO:     Started server process [XXXX]
INFO:     Uvicorn running on http://0.0.0.0:7000
```

**Test in a separate terminal:**

```bash
curl http://172.16.23.129:7000/latest
```

Expected:
```json
{"bandwidth_mbps": 18.45}
```

> If `ogstun` is not found, the server returns `0.1 Mbps` as a safe minimum. This happens if Open5GS is not running or no UE is attached.

## Running Both Servers Together

In production, run both servers simultaneously in separate terminals or using `&`:

```bash
# Terminal 1 — Throughput server:
cd ~/Desktop/MILP
uvicorn throughput_server:app --host 0.0.0.0 --port 7000

# Terminal 2 — MILP Gateway:
cd ~/Desktop/MILP
python3 milp_gateway.py
```

Or run both in the background:

```bash
cd ~/Desktop/MILP
nohup uvicorn throughput_server:app --host 0.0.0.0 --port 7000 > throughput.log 2>&1 &
nohup python3 milp_gateway.py > milp_gateway.log 2>&1 &

# Confirm both are running:
ps aux | grep -E "throughput_server|milp_gateway"
```

## Troubleshooting

### `ModuleNotFoundError: No module named 'pyomo'`

```bash
pip3 install pyomo
python3 -c "import pyomo; print(pyomo.__version__)"
```

### MILP solver not found at runtime

```bash
# Symptom in logs:
# WARNING: Could not locate the 'highs' executable

# Fix — install CBC as fallback:
sudo apt install -y coinor-cbc
which cbc    # must return /usr/bin/cbc

# The gateway tries HiGHS first, then falls back to CBC automatically
```

### Throughput server returns `0.1 Mbps` always

```bash
# Cause: ogstun interface not found (Open5GS not running or no UE attached)

# Check if ogstun exists:
ip a | grep ogstun

# Fix — ensure Open5GS core is running:
sudo systemctl status open5gs-upfd
sudo systemctl restart open5gs-upfd
```

### Edge nodes show `util: 0.95` (all assumed busy)

```bash
# Cause: Gateway cannot reach edge /metrics endpoints

# Test reachability:
curl http://172.16.23.129:30081/metrics

# Fix — confirm edge pods are running:
microk8s kubectl get pods | grep edge

# Fix — confirm MicroK8s VM IP is correct in milp_gateway.py:
# EDGE_SERVICES list should use 172.16.23.129
```

### Port 6001 already in use

```bash
sudo kill $(sudo lsof -t -i:6001)
python3 milp_gateway.py
```
