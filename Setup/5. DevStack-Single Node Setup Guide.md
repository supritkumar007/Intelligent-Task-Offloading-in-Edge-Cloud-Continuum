# OpenStack DevStack — Single Node Setup Guide
> **Platform:** Ubuntu 22.04 LTS · Single Node · OpenStack 2023.1 (Antelope)

---


> **Important:** Run all commands as your **regular sudo user**, NOT as root. DevStack creates and uses the `stack` user internally.

---

## Step 1 — Update System

```bash
sudo apt update && sudo apt -y upgrade
sudo apt install -y git vim wget curl net-tools
```

**Verify your machine IP (note it down — you'll need it for `HOST_IP`):**

```bash
ip a | grep inet
# or
hostname -I
```

---

## Step 2 — Create the `stack` User

DevStack requires a dedicated user with passwordless sudo.

```bash
sudo useradd -s /bin/bash -d /opt/stack -m stack
echo "stack ALL=(ALL) NOPASSWD: ALL" | sudo tee /etc/sudoers.d/stack
```

**Set up the stack directory:**

```bash
sudo mkdir -p /opt/stack
sudo chown stack:stack /opt/stack
```

**Verify the user was created correctly:**

```bash
id stack
# Expected: uid=... gid=... groups=...
ls -ld /opt/stack
# Expected: drwxr-xr-x ... stack stack ...
```

---

## Step 3 — Switch to the `stack` User

```bash
sudo su - stack
```

> All remaining steps are run as the `stack` user.

---

## Step 4 — Install Required Packages

```bash
sudo apt install -y git vim wget curl python3-pip
```

---

## Step 5 — Clone DevStack

```bash
git clone https://opendev.org/openstack/devstack
cd devstack
```

**Checkout the stable 2023.1 (Antelope) branch:**

```bash
git checkout -b stable-2023.1 origin/unmaintained/2023.1
```

**Verify the branch:**

```bash
git branch
# Expected: * stable-2023.1
```

---

## Step 6 — Create `local.conf`


Navigate into the samples directory first:

```bash
cd samples
```

Then create the config file:

> Replace `HOST_IP` with your **actual machine IP** from Step 1.
```bash
cat > local.conf <<'EOF'
[[local|localrc]]

# ─── Network ──────────────────────────────────────────────
HOST_IP=172.16.23.128          # ← CHANGE THIS to your machine IP
FLOATING_RANGE=172.16.23.0/24  # ← Should match your network subnet
FIXED_RANGE=10.11.12.0/24      # Internal VM network (leave as-is)

# ─── Passwords ────────────────────────────────────────────
ADMIN_PASSWORD=supersecret
DATABASE_PASSWORD=iheartdatabases
RABBIT_PASSWORD=flopsymopsy
SERVICE_PASSWORD=iheartksl

# ─── Networking Backend ───────────────────────────────────
Q_AGENT=ovn

# ─── Logging ──────────────────────────────────────────────
LOGFILE=/opt/stack/logs/stack.sh.log
LOGDAYS=2

# ─── Options ──────────────────────────────────────────────
IP_VERSION=4
EOF
```

**Double-check HOST_IP before proceeding:**

```bash
cat local.conf | grep HOST_IP
```

---

## Step 7 — Copy `local.conf` and Run DevStack

Copy the config up to the devstack root and start the installation:

```bash
cp local.conf ../
cd ..
```

> This step takes **30–60 minutes** depending on internet speed. Do not interrupt it.

```bash
FORCE=yes ./stack.sh
```

**Monitor the log in a separate terminal (optional but recommended):**

```bash
# Open another terminal, switch to stack user, then:
tail -f /opt/stack/logs/stack.sh.log
```

### What to expect

The script will:
1. Install all OpenStack services (Nova, Neutron, Glance, Keystone, Cinder, Horizon, etc.)
2. Configure OVN networking
3. Start all services
4. Print the Horizon dashboard URL at the end

### Successful completion output

```
=========================
DevStack Component Timing
=========================
...
This is your host IP address: 172.16.23.128
Keystone is serving at http://172.16.23.128/identity/
Horizon is available at http://172.16.23.128/dashboard
The default users are: admin and demo
The password: supersecret
```

## Step 8 — Access the Horizon Dashboard

Open a browser and navigate to:

```
http://<HOST_IP>/dashboard
```

**Login credentials:**

| Field | Value |
|-------|-------|
| Domain | `Default` |
| Username | `admin` |
| Password | `supersecret` (your `ADMIN_PASSWORD`) |

> After login, you are on the **Admin** panel. You can manage networks, images, flavors, VMs, and users from here.

## Useful Commands

### Service Management

```bash
# Check all DevStack screen sessions (services run in screen):
screen -ls

# Attach to the screen session to see live service output:
screen -r stack

# Inside screen: switch between windows with Ctrl+A then window number
# Detach from screen: Ctrl+A then D

# Restart all DevStack services (if something breaks):
cd ~/devstack && ./rejoin-stack.sh
```

### OpenStack CLI — Load Credentials First

```bash
source ~/devstack/openrc admin admin
```

> Run this every time you open a new terminal session before using `openstack` CLI commands.

```bash
# Verify credentials are loaded:
openstack token issue

# List all services:
openstack service list

# Check all endpoints:
openstack endpoint list
```

### Service Health Check

```bash
# Check Nova (compute) service:
openstack compute service list

# Check Neutron (network) agents:
openstack network agent list

# Check Cinder (block storage):
openstack volume service list

# Check all running processes:
sudo systemctl list-units | grep devstack
```

## When to Use `unstack.sh` and `clean.sh`

These two scripts are your reset tools. Know when to use each one:

| Script | What it does | When to use it |
|--------|-------------|----------------|
| `./unstack.sh` | Stops all running OpenStack services and screen sessions | Services are broken / hung · You want a soft reset without losing config |
| `./clean.sh` | Removes installed packages, databases, config files, and virtual environments | `stack.sh` failed midway · You want a full clean slate before re-running |

> **Order matters:** Always run `unstack.sh` before `clean.sh`. Never run `clean.sh` on a running stack.

### Soft Reset (services broken, config intact)

```bash
cd ~/devstack
./unstack.sh

# Then re-run:
FORCE=yes ./stack.sh
```

### Full Clean Reset (start completely fresh)

```bash
cd ~/devstack
./unstack.sh    # Stop everything first
./clean.sh      # Wipe all DevStack-installed state

# Then re-run:
FORCE=yes ./stack.sh
```

> After `clean.sh`, your `local.conf` is preserved — you do **not** need to recreate it.

## Debugging Common Errors

### stack.sh fails midway

```bash
# View the last 100 lines of the log:
tail -100 /opt/stack/logs/stack.sh.log

# Search for errors:
grep -i "error\|fail\|fatal" /opt/stack/logs/stack.sh.log | tail -30

# Clean up and re-run:
cd ~/devstack
./unstack.sh
./clean.sh
FORCE=yes ./stack.sh
```

### Horizon dashboard not loading

```bash
# Check if Apache is running:
sudo systemctl status apache2

# Restart Apache:
sudo systemctl restart apache2

# Check Horizon config:
cat /etc/openstack-dashboard/local_settings.py | grep ALLOWED_HOSTS
```

### Services not starting after reboot

DevStack does **not** survive reboots by default. After a reboot:

```bash
sudo su - stack
cd ~/devstack
./rejoin-stack.sh
```

> To make services survive reboots, use `systemd` wrappers — outside the scope of this guide.

### Neutron / OVN networking issues

```bash
# Check OVN status:
sudo ovs-vsctl show
sudo ovn-nbctl show
sudo ovn-sbctl show

# Check Neutron agents:
source ~/devstack/openrc admin admin
openstack network agent list
# All agents should show: Alive = :-)

# Restart Neutron services:
sudo systemctl restart devstack@q-svc
```

### Cannot reach floating IPs

```bash
# Re-enable IP forwarding on the host:
sudo sysctl -w net.ipv4.ip_forward=1

# Check iptables:
sudo iptables -t nat -L -n -v | grep MASQUERADE
```

### MySQL / Database errors

```bash
# Check MySQL status:
sudo systemctl status mysql

# Restart MySQL:
sudo systemctl restart mysql

# View MySQL error log:
sudo tail -50 /var/log/mysql/error.log
```
