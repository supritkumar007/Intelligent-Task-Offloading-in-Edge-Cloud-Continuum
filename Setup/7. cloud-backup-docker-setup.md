# Cloud Backup App Deployment Guide
> **Role:** Backup inference server — handles image classification requests when all edge nodes are overloaded or unavailable. Deployed via Docker on the OpenStack DevStack host machine.

---

## Architecture Overview

```
AI Scheduler (MicroK8s VM)
         |
         | All edge nodes overloaded?
         ▼
  Cloud Backup VM (DevStack Host)
  ┌──────────────────────────────┐
  │  Docker Container            │
  │  amogh0709/fastapi-app:v5    │
  │  CIFAR-10 Inference          │
  │  Host Port: 8000             │
  └──────────────────────────────┘
         |
         ▼
  POST /predict → returns classification result
  POST /data    → forwards result to Storage VM (172.16.23.128:8080)
```

### Key Parameters

| Parameter | Value |
|-----------|-------|
| Docker Image | `amogh0709/fastapi-application:v5` |
| Container Port | `8000` |
| Host Port | `8000` |
| Deployment Method | Docker (single container) |
| Host Machine | DevStack Host (OpenStack server) |

> **Alternative:** This can also be deployed on MicroK8s for auto-scaling. Docker is used here for simplicity.

## Part 1 — Install Docker

### 1.1 Update System

```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 Install Prerequisites

```bash
sudo apt install -y ca-certificates curl gnupg lsb-release
```

### 1.3 Add Docker's Official GPG Key

```bash
sudo mkdir -p /etc/apt/keyrings

curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
```

### 1.4 Add Docker Repository

```bash
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
```

### 1.5 Install Docker Engine

```bash
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin
```

### 1.6 Verify Installation

```bash
sudo docker --version
```

Expected output:
```
Docker version 27.x.x, build xxxxxxx
```

### 1.7 Run Docker Without `sudo` (Recommended)

```bash
sudo usermod -aG docker $USER
newgrp docker
```

### 1.8 Smoke Test

```bash
docker run hello-world
```

Expected output:
```
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

## Part 2 — Run the Backup Inference Container

### 2.1 Pull and Start the Container

```bash
docker run -d \
  --name cloud-backup \
  --restart unless-stopped \
  -p 8000:8000 \
  amogh0709/fastapi-application:v5
```

| Flag | Purpose |
|------|---------|
| `-d` | Run in detached (background) mode |
| `--name cloud-backup` | Give the container a readable name |
| `--restart unless-stopped` | Auto-restart on crash or host reboot |
| `-p 8000:8000` | Map host port 8000 → container port 8000 |

### 2.2 Verify the Container is Running

```bash
docker ps
```

Expected output:
```
CONTAINER ID   IMAGE                              STATUS         PORTS                    NAMES
xxxxxxxxxxxx   amogh0709/fastapi-application:v5   Up X seconds   0.0.0.0:8000->8000/tcp   cloud-backup
```

### 2.3 Check Startup Logs

```bash
docker logs cloud-backup
```

Expected output:
```
INFO:     Started server process [1]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

## Part 3 — Test the Container

### Health Check

```bash
curl http://localhost:8000/
```

### Send a Test Image for Classification

```bash
# Create a dummy image if you don't have one:
curl -o test.jpg https://upload.wikimedia.org/wikipedia/commons/thumb/3/3a/Cat03.jpg/320px-Cat03.jpg

# Send it to the backup inference server:
curl -X POST http://localhost:8000/predict \
  -F "file=@test.jpg"
```

Expected response:
```json
{
  "prediction": "cat",
  "confidence": 0.91,
  "class_id": 3
}
```

### Test from the Scheduler / MicroK8s VM

```bash
# Replace with your DevStack host IP:
curl -X POST http://172.16.23.100:8000/predict \
  -F "file=@test.jpg"
```

> This confirms the scheduler can reach the backup server over the network using `CLOUD_SERVER_URL=http://172.16.23.100:8000`.

## Troubleshooting

### Port 8000 not reachable from MicroK8s VM

```bash
# Diagnosis — confirm port is bound:
sudo ss -tlnp | grep 8000

# Diagnosis — check firewall:
sudo ufw status
sudo ufw allow 8000/tcp

# Diagnosis — test from DevStack host itself:
curl http://localhost:8000/predict -X POST -F "file=@test.jpg"
```

### Image pull fails (`docker run` hangs or errors)

```bash
# Pull image separately with progress:
docker pull amogh0709/fastapi-application:v5

# If behind a proxy, configure Docker daemon proxy:
sudo mkdir -p /etc/systemd/system/docker.service.d
sudo bash -c 'cat > /etc/systemd/system/docker.service.d/proxy.conf <<EOF
[Service]
Environment="HTTP_PROXY=http://proxy:port"
Environment="HTTPS_PROXY=http://proxy:port"
EOF'
sudo systemctl daemon-reload && sudo systemctl restart docker
```

### `Permission denied` running docker without sudo

```bash
# Re-apply group membership:
sudo usermod -aG docker $USER
newgrp docker

# Verify:
groups | grep docker
```

## Quick Cheat Sheet

```bash
# Install Docker:
sudo apt install -y docker-ce docker-ce-cli containerd.io

# Run container:
docker run -d --name cloud-backup --restart unless-stopped \
  -p 8000:8000 amogh0709/fastapi-application:v5

# Check status:
docker ps

# Live logs:
docker logs -f cloud-backup

# Test locally:
curl -X POST http://localhost:8000/predict -F "file=@test.jpg"

# Test from remote:
curl -X POST http://172.16.23.100:8000/predict -F "file=@test.jpg"

# Restart:
docker restart cloud-backup

# Remove and redeploy:
docker rm -f cloud-backup && docker run -d --name cloud-backup \
  --restart unless-stopped -p 8000:8000 amogh0709/fastapi-application:v5
```
