## Part 1 — Install MicroK8s

### 1.1 Update System

```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 Install Snapd and MicroK8s

```bash
sudo apt install snapd -y
sudo snap install microk8s --classic
```

### 1.3 Configure User Permissions

```bash
sudo usermod -a -G microk8s $USER
sudo chown -f -R $USER ~/.kube

# Apply group changes — log out and back in, or run:
newgrp microk8s
```

### 1.4 Wait for MicroK8s to Be Ready

```bash
microk8s status --wait-ready
```

Expected output:
```
microk8s is running
high-availability: no
...
```

### 1.5 Enable Required Add-ons

```bash
microk8s enable dns dashboard storage ingress
microk8s enable metrics-server registry hostpath-storage
```

> These may take 1–2 minutes each. Wait for each to complete before proceeding.

### 1.6 Verify Node is Ready

```bash
microk8s kubectl get nodes
```

Expected output:
```
NAME       STATUS   ROLES    AGE   VERSION
<hostname> Ready    <none>   Xm    v1.xx.x
```

> Status must be `Ready` before deploying pods. If it shows `NotReady`, wait 30 seconds and retry.

## Part 2 — Configure kubectl Alias (Optional but Recommended)

Avoid typing `microk8s kubectl` every time:

```bash
echo "alias kubectl='microk8s kubectl'" >> ~/.bashrc
source ~/.bashrc
```

Or use the built-in snap alias:

```bash
sudo snap alias microk8s.kubectl kubectl
```

## Part 4 — Deploy Edge Nodes

### 4.1 Apply the Deployment

```bash
microk8s kubectl apply -f edge-deployment.yaml
```

Expected output:
```
deployment.apps/edge-node1 created
service/edge-service1 created
deployment.apps/edge-node2 created
service/edge-service2 created
...
```

### 4.2 Watch Pods Come Up

```bash
microk8s kubectl get pods -w
```

Wait until all 9 pods show `Running` (may take 1–3 minutes for image pull on first deploy):

```
NAME                          READY   STATUS    RESTARTS   AGE
edge-node1-xxxxxxxxx-xxxxx    1/1     Running   0          30s
edge-node2-xxxxxxxxx-xxxxx    1/1     Running   0          30s
...
edge-node9-xxxxxxxxx-xxxxx    1/1     Running   0          45s
```

> If pods stay in `ContainerCreating`, the image is still being pulled. Wait and re-check.


### Test an Edge Node Endpoint

```bash
# Replace <VM-IP> with your Open5GS + MicroK8s VM IP
curl http://<VM-IP>:30081/

# Or test from inside the VM:
curl http://localhost:30081/
```

### Check Deployments

```bash
microk8s kubectl get deployments
```

## Useful Commands

### Pod Management

```bash
# Get all pods with node and IP info:
microk8s kubectl get pods -o wide

# Describe a specific pod (useful for debugging):
microk8s kubectl describe pod <pod-name>

# View logs of a pod:
microk8s kubectl logs <pod-name>

# Stream live logs:
microk8s kubectl logs -f <pod-name>

# Exec into a running pod:
microk8s kubectl exec -it <pod-name> -- /bin/bash
```

### Deployment Management

```bash
# Restart all edge node deployments (rolling restart):
for i in $(seq 1 9); do
  microk8s kubectl rollout restart deployment/edge-node$i
done

# Check rollout status:
microk8s kubectl rollout status deployment/edge-node1

# Scale a deployment (e.g., 2 replicas):
microk8s kubectl scale deployment edge-node1 --replicas=2
```

### Update Image Version

```bash
# Update all edge nodes to a new image version (e.g., v6):
for i in $(seq 1 9); do
  microk8s kubectl set image deployment/edge-node$i edge=amogh0709/fastapi-application:v6
done
```

### Cluster Health

```bash
# Node status:
microk8s kubectl get nodes

# All resources at once:
microk8s kubectl get all

# Resource usage (requires metrics-server):
microk8s kubectl top pods
microk8s kubectl top nodes
```

## Troubleshooting

### Pods in `ImagePullBackOff` or `ErrImagePull`

```bash
# Check pod events:
microk8s kubectl describe pod <pod-name> | grep -A10 Events

# Verify image name is correct:
# amogh0709/fastapi-application:v5

# Check internet access from VM:
curl https://registry-1.docker.io/v2/

# If behind a proxy, configure containerd proxy settings
```

### Pods in `CrashLoopBackOff`

```bash
# View crash logs:
microk8s kubectl logs <pod-name> --previous

# Check if REMOTE_SERVER_URL is reachable from inside the cluster:
microk8s kubectl exec -it <pod-name> -- curl http://172.16.23.128:8080/data
```

### NodePort Not Accessible from Outside

```bash
# Verify the port is actually bound:
sudo ss -tlnp | grep 3008

# Check if firewall is blocking:
sudo ufw status
sudo ufw allow 30081:30090/tcp

# Verify service is correctly mapped:
microk8s kubectl get svc edge-service1 -o yaml | grep nodePort
```

### MicroK8s DNS Not Working (Pods Can't Resolve Hostnames)

```bash
# Check CoreDNS pods:
microk8s kubectl get pods -n kube-system | grep dns

# Restart DNS:
microk8s disable dns && microk8s enable dns
```

### Delete and Redeploy (Full Reset)

```bash
# Delete all edge deployments and services:
microk8s kubectl delete -f edge-deployment.yaml

# Verify everything is removed:
microk8s kubectl get pods
microk8s kubectl get services

# Redeploy:
microk8s kubectl apply -f edge-deployment.yaml
```
