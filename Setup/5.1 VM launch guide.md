# OpenStack DevStack — Launch an Ubuntu VM Guide
> **Prerequisite:** DevStack is installed and running · Horizon dashboard is accessible

---

## Before You Begin — Load OpenStack Credentials

You have two ways to get the credentials file. **Option A** uses the file DevStack already created for you. **Option B** shows how to download it directly from the Horizon dashboard.

---

### Option A — Use the DevStack `openrc` File (Quickest)

DevStack automatically generates this file during installation:

```bash
source ~/devstack/openrc admin admin
```

---

### Option B — Download `openrc` from Horizon Dashboard

Use this when you don't have direct access to the DevStack host filesystem, or you want the file scoped to a specific project.

**1. Open the Horizon dashboard in your browser:**

```
http://<HOST_IP>/dashboard
```

**2. Log in as admin:**

| Field | Value |
|-------|-------|
| Domain | `Default` |
| Username | `admin` |
| Password | `supersecret` |

**3. In the top navigation bar, click on the project name dropdown (top-left, next to the OpenStack logo) — make sure you are in the `admin` project.**

**4. Go to:**

```
Top-right corner → click "admin" (your username) → Project Settings
```

Or navigate directly via the left sidebar:

```
Project → API Access
```

**5. On the API Access page, click the `Download OpenStack RC File` button (top-right of the page) → select `OpenStack RC File`.**

This downloads a file named `admin-openrc.sh`.

**6. Copy the file to your DevStack host and source it:**

```bash
# If downloaded to your local machine, scp it to the host:
scp admin-openrc.sh stack@<HOST_IP>:~/

# On the DevStack host, source it:
source ~/admin-openrc.sh
```

It will prompt you for the admin password:

```
Please enter your OpenStack Password for project admin as user admin:
```

Enter `supersecret` (your `ADMIN_PASSWORD` from `local.conf`).

---

### Verify Credentials Are Loaded (Both Options)

```bash
openstack token issue
# Expected: a table showing token ID, expiry, project, user
```

If you see a table output, credentials are loaded correctly. If you get an authentication error, double-check the password or re-source the file.

> **Tip:** You must run `source` again every time you open a new terminal session. The environment variables it sets do not persist across sessions.

## Step 1 — Verify OpenStack Services Are Running

Before creating anything, confirm all core services are healthy:

```bash
openstack compute service list     # Nova — all should be "up"
openstack network agent list       # Neutron/OVN — all "Alive: :-)"
openstack image list               # Glance — check existing images
openstack flavor list              # Nova — check existing flavors
openstack network list             # Neutron — check existing networks
openstack security group list      # Check existing security groups
```

---

## Step 2 — Download Ubuntu Cloud Image

```bash
cd ~/Downloads

wget https://cloud-images.ubuntu.com/jammy/current/jammy-server-cloudimg-amd64.img
```

**Verify the download:**

```bash
ls -lh ~/Downloads/jammy-server-cloudimg-amd64.img
# Expected: ~600 MB file

file ~/Downloads/jammy-server-cloudimg-amd64.img
# Expected: QEMU QCOW Image (v3)...
```

---

## Step 3 — Upload Ubuntu Image to Glance

```bash
openstack image create "Ubuntu-22.04" \
  --file ~/Downloads/jammy-server-cloudimg-amd64.img \
  --disk-format qcow2 \
  --container-format bare \
  --public
```

**Verify the image is active:**

```bash
openstack image list
# Status should show: active
```

> If status stays `queued`, Glance may have a storage issue — check `/opt/stack/logs/g-api.log`.

---

## Step 4 — Create a Flavor (VM Size)

```bash
openstack flavor create \
  --vcpus 1 \
  --ram 1024 \
  --disk 10 \
  small
```

**List all available flavors:**

```bash
openstack flavor list
```

> DevStack may already have default flavors (`m1.tiny`, `m1.small`, etc.). You can skip this step and use an existing one.

---

## Step 5 — Create Private Network

```bash
openstack network create private-net

openstack subnet create private-subnet \
  --network private-net \
  --subnet-range 10.0.0.0/24 \
  --dns-nameserver 8.8.8.8
```

**Verify:**

```bash
openstack network list
openstack subnet list
```

---

## Step 6 — Create Router and Connect to External Network

```bash
openstack router create router1

# Find the external network name (usually 'public'):
openstack network list --external

# Set external gateway:
openstack router set router1 --external-gateway public

# Connect private subnet to the router:
openstack router add subnet router1 private-subnet
```

**Verify routing:**

```bash
openstack router show router1
# Should show: external_gateway_info with public network
```

---

## Step 7 — Configure Security Group Rules

Allow SSH and ICMP (ping) on the default security group:

```bash
# Allow SSH (port 22):
openstack security group rule create \
  --proto tcp --dst-port 22 default

# Allow ICMP (ping):
openstack security group rule create \
  --proto icmp default
```

**Verify rules are added:**

```bash
openstack security group rule list default
```

---

## Step 8 — Create SSH Keypair

```bash
mkdir -p ~/.ssh

openstack keypair create devstack_key > ~/.ssh/devstack_key.pem

chmod 600 ~/.ssh/devstack_key.pem
```

**Verify the keypair:**

```bash
openstack keypair list
# Should show: devstack_key

# Check the private key file:
ls -l ~/.ssh/devstack_key.pem
# Should show: -rw------- (permissions 600)
```

---

## Step 9 — Get Network ID

```bash
openstack network list
```

Example output:
```
+--------------------------------------+-------------+
| ID                                   | Name        |
+--------------------------------------+-------------+
| a60336d5-bce3-46ac-84ef-2ce69b4b36f9 | private-net |
| ...                                  | public      |
+--------------------------------------+-------------+
```

Set the private network ID as a variable:

```bash
NET_ID=$(openstack network show private-net -f value -c id)
echo $NET_ID   # Confirm it printed the UUID
```

---

## Step 10 — Get Security Group ID

```bash
openstack security group list
```

Set the default security group ID as a variable:

```bash
SG_ID=$(openstack security group show default -f value -c id)
echo $SG_ID   # Confirm it printed the UUID
```

---

## Step 11 — Launch the Ubuntu VM

```bash
openstack server create ubuntu-vm \
  --flavor small \
  --image "Ubuntu-22.04" \
  --nic net-id=$NET_ID \
  --security-group $SG_ID \
  --key-name devstack_key
```

**Monitor VM status:**

```bash
# Watch until status becomes ACTIVE:
watch openstack server list

# Or check once:
openstack server show ubuntu-vm
```

> Status transitions: `BUILD` → `ACTIVE` (takes ~30–60 seconds)
> If it goes to `ERROR`, see the Troubleshooting section below.

---

## Step 12 — Assign a Floating IP

A floating IP lets you SSH into the VM from the host machine.

```bash
# Allocate a floating IP from the public pool:
FLOAT_IP=$(openstack floating ip create public -f value -c floating_ip_address)
echo "Floating IP: $FLOAT_IP"

# Attach it to your VM:
openstack server add floating ip ubuntu-vm $FLOAT_IP
```

**Verify:**

```bash
openstack server show ubuntu-vm | grep addresses
# Should show both the private IP (10.0.0.x) and floating IP
```

---

## Step 13 — Fix Internet Connectivity (Run on Host Machine)

> Run these commands on the **OpenStack host** (not inside the VM).

```bash
# Enable IP forwarding:
sudo sysctl -w net.ipv4.ip_forward=1

# Add NAT masquerade (replace ens33 with your internet-facing interface):
sudo iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE

# Allow forwarding between interfaces:
sudo iptables -A FORWARD -i ens33 -o br-ex -m state --state RELATED,ESTABLISHED -j ACCEPT
sudo iptables -A FORWARD -i br-ex -o ens33 -j ACCEPT
```

**Find your internet-facing interface name:**

```bash
ip route | grep default
# Example: default via 172.16.23.2 dev ens33  ← use ens33
```

---

## Step 14 — SSH into the VM

```bash
ssh -i ~/.ssh/devstack_key.pem ubuntu@$FLOAT_IP
```

> Default username for Ubuntu cloud images is `ubuntu`.

**First-time SSH tip:** If you see a host key warning, type `yes` to accept.

---

## Step 15 — Fix DNS Inside the VM

Run these commands **inside the Ubuntu VM** after SSH-ing in:

```bash
# Set DNS servers:
sudo bash -c 'cat > /etc/resolv.conf <<EOF
nameserver 8.8.8.8
nameserver 1.1.1.1
EOF'

# Fix hostname resolution:
sudo bash -c 'echo "127.0.1.1 ubuntu-vm" >> /etc/hosts'
```

---

## Step 16 — Verify Internet Connectivity

Run inside the VM:

```bash
ping -c 3 8.8.8.8         # Test IP-level connectivity
ping -c 3 google.com      # Test DNS resolution
sudo apt update            # Test package manager access
```

**Expected output for ping:**
```
3 packets transmitted, 3 received, 0% packet loss
```

## VM Management Commands

```bash
# List all VMs and their status:
openstack server list

# Start / Stop / Reboot:
openstack server stop ubuntu-vm
openstack server start ubuntu-vm
openstack server reboot ubuntu-vm

# View VM console log (useful if SSH fails):
openstack console log show ubuntu-vm

# Get VNC console URL (browser-based terminal):
openstack console url show ubuntu-vm

# Delete VM:
openstack server delete ubuntu-vm
```

## Troubleshooting

### VM stuck in `BUILD` or goes to `ERROR`

```bash
# View error details:
openstack server show ubuntu-vm | grep -E "status|fault"

# View full Nova compute log:
sudo tail -100 /opt/stack/logs/n-cpu.log | grep -i error

# View VM console output (useful for boot errors):
openstack console log show ubuntu-vm
```

### Cannot SSH into VM

```bash
# Check floating IP is assigned:
openstack server show ubuntu-vm | grep addresses

# Confirm security group allows port 22:
openstack security group rule list default | grep 22

# Test if VM is reachable:
ping $FLOAT_IP

# Check SSH key permissions:
ls -l ~/.ssh/devstack_key.pem
# Must be: -rw------- (600)
```

### VM has no internet inside

```bash
# On the host — verify IP forwarding:
cat /proc/sys/net/ipv4/ip_forward
# Must be: 1

# On the host — verify NAT rule:
sudo iptables -t nat -L POSTROUTING -n -v | grep MASQUERADE

# Inside the VM — check default route:
ip route
# Should show: default via 10.0.0.1
```

### Floating IP not reachable from host

```bash
# Check br-ex interface exists:
ip a | grep br-ex

# Check OVN north/south connectivity:
sudo ovn-nbctl show | grep -A5 router1

# Re-add iptables rules (they reset on reboot):
sudo iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE
sudo iptables -I FORWARD 1 -j ACCEPT
```

### Image upload fails or stays `queued`

```bash
# Check Glance API log:
sudo tail -50 /opt/stack/logs/g-api.log | grep -i error

# Check available disk space:
df -h /opt/stack

# Re-upload with verbose output:
openstack image create "Ubuntu-22.04" \
  --file ~/Downloads/jammy-server-cloudimg-amd64.img \
  --disk-format qcow2 \
  --container-format bare \
  --public \
  --debug
```

## Quick Setup Cheat Sheet

```bash
# 1. Load credentials:
source ~/devstack/openrc admin admin

# 2. Set IDs as variables:
NET_ID=$(openstack network show private-net -f value -c id)
SG_ID=$(openstack security group show default -f value -c id)

# 3. Launch VM:
openstack server create ubuntu-vm \
  --flavor small --image "Ubuntu-22.04" \
  --nic net-id=$NET_ID --security-group $SG_ID \
  --key-name devstack_key

# 4. Get floating IP:
FLOAT_IP=$(openstack floating ip create public -f value -c floating_ip_address)
openstack server add floating ip ubuntu-vm $FLOAT_IP

# 5. SSH in:
ssh -i ~/.ssh/devstack_key.pem ubuntu@$FLOAT_IP
```
