# Private 5G SA Network Setup Guide

> **Architecture:** gNB (srsRAN) runs on the **localhost machine** · Open5GS 5G Core runs on a **separate VM**

---

## System Architecture

```
UE (5G Phone)
      |
 USRP B210 (USB 3.0)
      |
 Localhost / gNB Machine  →  IP: 172.16.23.1   (srsRAN)
      |
  [N2 / N3 Network — 172.16.23.0/24]
      |
 Core VM                  →  IP: 172.16.23.129  (Open5GS)
      |
  Internet
```

---

## Components

| Layer | Component | Host |
|-------|-----------|------|
| Core (5GC) | Open5GS | Core VM — `172.16.23.129` |
| RAN | srsRAN Project | Localhost — `172.16.23.1` |
| SDR | USRP B210 | USB 3.0 on localhost |
| SIM | Open-Cells UICC | USB card reader on localhost |
| DB | MongoDB | Core VM |

**Hardware requirements (both machines):** Ubuntu 22.04 LTS · Intel i7, 32 GB RAM recommended

**Key RF parameters:**

| Parameter | Value |
|-----------|-------|
| PLMN | `99970` |
| TAC | `7` |
| PCI | `1` |
| Band | n78 |
| DL ARFCN | `632628` |
| Bandwidth | 20 MHz |
| SCS | 30 kHz |

## Part 1 — gNB Setup (Localhost Machine)

### 1.1 Install Dependencies

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y cmake make gcc g++ pkg-config \
    libfftw3-dev libmbedtls-dev libsctp-dev \
    libyaml-cpp-dev libgtest-dev libuhd-dev uhd-host
```

### 1.2 USRP B210 Setup

Install UHD firmware images and verify the device:

```bash
sudo python3 /usr/lib/uhd/utils/uhd_images_downloader.py

# Connect USRP B210 via USB 3.0, then verify:
uhd_find_devices      # Should list the B210
uhd_usrp_probe        # Displays detailed device info
```

> **Troubleshooting:** If not detected, reconnect to a USB 3.0 port and retry. Reinstall drivers if needed:
> `sudo apt install --reinstall libuhd-dev uhd-host`

### 1.3 Build srsRAN

```bash
git clone https://github.com/srsran/srsRAN_Project.git
cd srsRAN_Project
mkdir build && cd build
cmake ../
make -j $(nproc)
sudo make install
```

### 1.4 Configure gNB

Edit the configuration file:

```bash
nano ~/Desktop/srsRAN_Project/configs/gnb_rf_b200_tdd_n78_20mhz.yml
```

**Full working configuration:**

```yaml
cu_cp:
  amf:
    addr: 172.16.23.129       # Core VM IP
    port: 38412
    bind_addr: 172.16.23.1    # Localhost gNB IP

ru_sdr:
  device_driver: uhd
  device_args: type=b200
  srate: 23.04
  otw_format: sc12
  tx_gain: 75
  rx_gain: 70

cell_cfg:
  dl_arfcn: 632628
  band: 78
  channel_bandwidth_MHz: 20
  common_scs: 30
  plmn: "99970"
  tac: 7
  pci: 1

log:
  filename: /tmp/gnb.log
  all_level: info
```

### 1.5 Run gNB

```bash
cd ~/Desktop/srsRAN_Project/build/apps/gnb
sudo ./gnb -c ~/Desktop/srsRAN_Project/configs/gnb_rf_b200_tdd_n78_20mhz.yml

# Monitor logs in a separate terminal:
sudo tail -f /tmp/gnb.log
```

> **Expected:** Logs should show a successful AMF connection to `172.16.23.129`.

## Part 2 — 5G Core Setup (Core VM — `172.16.23.129`)

### 2.1 Install MongoDB

```bash
sudo apt update
sudo apt install -y gnupg curl

curl -fsSL https://pgp.mongodb.com/server-8.0.asc \
  | sudo gpg -o /usr/share/keyrings/mongodb-server-8.0.gpg --dearmor

echo "deb [arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-8.0.gpg] \
https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/8.0 multiverse" \
  | sudo tee /etc/apt/sources.list.d/mongodb-org-8.0.list

sudo apt update
sudo apt install -y mongodb-org
sudo systemctl start mongod
sudo systemctl enable mongod

# Verify:
sudo systemctl status mongod   # Should show: active (running)
```

### 2.2 Install Open5GS

```bash
sudo add-apt-repository ppa:open5gs/latest
sudo apt update
sudo apt install -y open5gs

# Verify all 17 services are active:
sudo systemctl status open5gs-* | grep -c "active"
```

### 2.3 Configure Open5GS — IP Addresses

Four files need to be updated with the correct VM IP.

#### AMF — `/etc/open5gs/amf.yaml`

```yaml
ngap:
  server:
    - address: 172.16.23.129
```

#### UPF — `/etc/open5gs/upf.yaml`

```yaml
upf:
  pfcp:
    server:
      - address: 127.0.0.7      # PFCP stays localhost
  gtpu:
    server:
      - address: 172.16.23.129  # GTP-U must use Core VM IP
```

#### SMF — `/etc/open5gs/smf.yaml`

```yaml
# Verify this is present — DO NOT change:
pfcp:
  client:
    upf:
      - address: 127.0.0.7
```

#### NRF — `/etc/open5gs/nrf.yaml`

```yaml
# Leave default — no changes needed:
sbi:
  server:
    - address: 127.0.0.10
```

### 2.4 Restart Open5GS

```bash
sudo systemctl restart open5gs-*
sudo systemctl status open5gs-* | grep -c "active"   # Expect: 17
```

### 2.5 NAT Configuration (Internet Access)

```bash
# Enable IP forwarding (temporary):
sudo sysctl -w net.ipv4.ip_forward=1

# Make it permanent:
echo "net.ipv4.ip_forward=1" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

# Find internet-facing interface:
ip route | grep default
# Example output: default via 172.16.23.2 dev ens33

# Add NAT masquerade (replace ens33 with your interface):
sudo iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE
sudo iptables -I FORWARD 1 -j ACCEPT

# Persist rules across reboots:
sudo apt install -y iptables-persistent
sudo dpkg-reconfigure iptables-persistent

# Verify:
sudo iptables -t nat -L POSTROUTING -n -v   # Should show MASQUERADE
```

### 2.6 WebUI — Add Subscriber

Install Node.js and the Open5GS WebUI:

```bash
sudo apt install -y ca-certificates curl gnupg
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://deb.nodesource.com/gpgkey/nodesource-repo.gpg.key \
  | sudo gpg --dearmor -o /etc/apt/keyrings/nodesource.gpg

NODE_MAJOR=20
echo "deb [signed-by=/etc/apt/keyrings/nodesource.gpg] \
https://deb.nodesource.com/node_$NODE_MAJOR.x nodistro main" \
  | sudo tee /etc/apt/sources.list.d/nodesource.list

sudo apt update && sudo apt install -y nodejs
curl -fsSL https://open5gs.org/open5gs/assets/webui/install | sudo -E bash -
```

Access the WebUI at `http://172.16.23.129:9999` (or `localhost:9999` from the VM):

| Field | Value |
|-------|-------|
| Username | `admin` |
| Password | `1423` |

Add a new subscriber with these values:

| Field | Value |
|-------|-------|
| IMSI | `999700000000001` |
| Key (Ki) | `6874736969202073796d4b2079650a73` |
| OPC | `504f20634f6320504f50206363500a4f` |
| APN | `internet` |
| IP Allocation | Dynamic |

> **Security:** Change the default WebUI password immediately after setup.

## Part 3 — SIM Programming (Localhost Machine)

### 3.1 Prerequisites

- Programmable SIM card + USB card reader
- [`uicc-v3.3.tgz`](https://open-cells.com/d5138782a8739209ec5760865b1e53b0/uicc-v3.3.tgz) downloaded from Open-Cells 

### 3.2 Compile UICC Tools

```bash
tar -xvzf uicc-v3.3.tgz
cd uicc-v3.3
make
```

### 3.3 Program the SIM

Insert the SIM card into the USB reader and run:

```bash
sudo ./program_uicc \
  --adm 12345678 \
  --imsi 999700000000001 \
  --isdn 00000001 \
  --acc 0001 \
  --key 6874736969202073796d4b2079650a73 \
  --opc 504f20634f6320504f50206363500a4f \
  --spn "CSE" \
  --authenticate \
  --noreadafter
```

**Verification commands:**

```bash
sudo ./program_uicc              # Read basic card data
sudo DEBUG=y ./program_uicc      # Enable detailed traces
sudo ./program_uicc --help       # View all parameters
```

## Part 4 — Verification

### Core Verification (on Core VM)

```bash
# Check PFCP association (SMF ↔ UPF):
sudo grep PFCP /var/log/open5gs/smf.log | tail
sudo grep PFCP /var/log/open5gs/upf.log | tail
# Expected: "PFCP associated"

# Check GTP-U is listening:
sudo ss -lunp | grep 2152
# Expected: 172.16.23.129:2152
```

### UE Verification (on Core VM)

```bash
# Check UE gets an IP from ogstun:
sudo tcpdump -i ogstun -n
# Expected: packets from 10.45.0.x

# Verify GTP tunnel between gNB and Core:
sudo tcpdump -i any -n udp port 2152
# Expected: 172.16.23.1 <-> 172.16.23.129
```

## Troubleshooting

### MongoDB not starting
```bash
sudo journalctl -u mongod
# Check: disk space, permissions under /var/lib/mongodb
```

### Open5GS services not active
```bash
sudo systemctl restart open5gs-*
# Check logs:
ls /var/log/open5gs/
```

### USRP B210 not detected
```bash
# Ensure USB 3.0 port, then:
sudo apt install --reinstall libuhd-dev uhd-host
sudo python3 /usr/lib/uhd/utils/uhd_images_downloader.py
```

### gNB cannot connect to AMF
```bash
# Test reachability from localhost:
ping 172.16.23.129

# Verify in gnb config:
#   addr: 172.16.23.129   ← must match Core VM IP
#   bind_addr: 172.16.23.1

# Check AMF config on Core VM:
cat /etc/open5gs/amf.yaml | grep -A3 ngap
```

### UE attached but no internet
```bash
# Re-add NAT rule (replace ens33 with your interface):
sudo iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE
```

### SMF — No associated UPF
```bash
# Check upf.yaml: pfcp.server.address must be 127.0.0.7
# Check smf.yaml: upf.address must be 127.0.0.7
sudo systemctl restart open5gs-upfd open5gs-smfd
```

### GTP-U address wrong (no data plane traffic)
```bash
# In upf.yaml, gtpu.server.address MUST be the Core VM IP, NOT localhost:
#   Wrong:   address: 127.0.0.7
#   Correct: address: 172.16.23.129
sudo systemctl restart open5gs-upfd
```

## Additional Considerations

**Security**
- Change WebUI default password (`admin/1423`) immediately after setup.
- Restrict WebUI access using a firewall (`sudo ufw allow from 172.16.23.0/24`).

**Performance**
- Monitor CPU and memory during gNB operation (`htop`).
- Ensure adequate cooling on the localhost machine.

**Regulatory Compliance**
- Confirm band n78 (ARFCN 632628) usage complies with local spectrum regulations.
- Obtain any required radio transmission licenses before operating.

**Testing**
- Use a 5G SA-capable device with the programmed SIM for end-to-end validation.
- Measure throughput with `iperf3` between the UE and a remote server.

**Backup**
```bash
mongodump --out ~/open5gs-backup-$(date +%F)
```
