# Cloud Storage VM Setup Guide
> **Role:** Receives processed images and inference results uploaded by edge nodes and the cloud backup node after classification. Runs a Node.js Express server on port `8080`.

---

## Architecture Overview

```
Edge Nodes (MicroK8s)          Cloud Backup VM
  edge-node1 … edge-node9   +  172.16.23.100:8000
         |                            |
         |   POST /data (multipart)   |
         └────────────┬───────────────┘
                      ▼
         Cloud Storage VM  ←── Floating IP assigned
         172.16.23.128:8080
              |
         ┌────▼──────────────────┐
         │  Node.js Express App  │
         │  POST /data           │
         │  Saves to ./uploads/  │
         └───────────────────────┘
```

### Key Parameters

| Parameter | Value |
|-----------|-------|
| VM OS | Ubuntu 22.04 (Jammy) |
| App Port | `8080` |
| Upload Endpoint | `POST http://<Floating-IP>:8080/data` |
| Storage Directory | `~/storage-app/uploads/` |
| Node.js Version | 20.x (LTS) |

## Part 1 — Install Node.js

### 1.1 Update System

```bash
sudo apt update && sudo apt upgrade -y
```

### 1.2 Install Node.js 20.x (LTS)

```bash
sudo apt install -y ca-certificates curl gnupg
sudo mkdir -p /etc/apt/keyrings

curl -fsSL https://deb.nodesource.com/gpgkey/nodesource-repo.gpg.key \
  | sudo gpg --dearmor -o /etc/apt/keyrings/nodesource.gpg

echo "deb [signed-by=/etc/apt/keyrings/nodesource.gpg] \
https://deb.nodesource.com/node_20.x nodistro main" \
  | sudo tee /etc/apt/sources.list.d/nodesource.list

sudo apt update
sudo apt install -y nodejs
```

### 1.3 Verify Installation

```bash
node -v
# Expected: v20.x.x

npm -v
# Expected: 10.x.x
```

## Part 2 — Set Up the Storage App

### 2.1 Create Project Directory

```bash
mkdir storage-app && cd storage-app
```

### 2.2 Initialise npm Project

```bash
npm init -y
```

Expected output:
```
Wrote to /home/ubuntu/storage-app/package.json
```

### 2.3 Install Dependencies

```bash
npm install express multer path
```

| Package | Purpose |
|---------|---------|
| `express` | HTTP server framework |
| `multer` | Multipart file upload middleware |
| `path` | Node.js built-in path utilities |

Expected output:
```
added 60 packages in 3s
```

### 2.4 Create the Server File

```bash
vim index.js
```

Paste the following:

```javascript
const express = require('express');
const multer = require('multer');
const path = require('path');
const fs = require('fs');

const app = express();
const PORT = 8080;

// Configure storage for uploaded files
const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    const uploadDir = 'uploads';
    if (!fs.existsSync(uploadDir)) {
      fs.mkdirSync(uploadDir);
    }
    cb(null, uploadDir);
  },
  filename: (req, file, cb) => {
    // Save with original name + timestamp to avoid collisions
    const timestamp = Date.now();
    const ext = path.extname(file.originalname);
    const basename = path.basename(file.originalname, ext);
    cb(null, `${basename}-${timestamp}${ext}`);
  }
});

const upload = multer({ storage });

app.post('/data', upload.single('file'), (req, res) => {
  if (!req.file) {
    return res.status(400).send('No file uploaded.');
  }
  console.log(`Received file: ${req.file.filename}`);
  res.status(200).send('File received successfully.');
});

app.listen(PORT, () => {
  console.log(`Express server listening on port ${PORT}`);
});
```

Save and exit: `Esc` → `:wq` → `Enter`

## Part 5 — Run as Background Service (Keep Running After SSH Exit)

### Option A — Using `nohup` (Quick)

```bash
cd ~/storage-app
nohup node index.js > server.log 2>&1 &
echo $! > server.pid

# Verify it's running:
cat server.pid
ps aux | grep node
```

To stop:
```bash
kill $(cat ~/storage-app/server.pid)
```

### Option B — Using `systemd` (Recommended for Persistent Restarts)

```bash
sudo bash -c 'cat > /etc/systemd/system/storage-app.service <<EOF
[Unit]
Description=Node.js Cloud Storage Server
After=network.target

[Service]
ExecStart=/usr/bin/node /home/ubuntu/storage-app/index.js
WorkingDirectory=/home/ubuntu/storage-app
Restart=always
User=ubuntu
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
EOF'

sudo systemctl daemon-reload
sudo systemctl enable storage-app
sudo systemctl start storage-app
```

**Verify:**

```bash
sudo systemctl status storage-app
# Should show: active (running)
```

## Troubleshooting

### Port 8080 not reachable from outside the VM

```bash
# Symptom: curl times out from DevStack host

# Diagnosis — check port is listening:
sudo ss -tlnp | grep 8080

# Diagnosis — check security group has port 8080 open:
# Horizon → Project → Network → Security Groups → default → Manage Rules

# Fix — re-add iptables rule on DevStack host if NAT is missing:
sudo iptables -t nat -A POSTROUTING -o ens33 -j MASQUERADE
```

### `EACCES` permission denied on port 8080

```bash
# Node.js can use ports > 1024 without sudo — 8080 is fine
# If you see this error, check if another process is using port 8080:
sudo ss -tlnp | grep 8080

# Kill the conflicting process:
sudo kill $(sudo lsof -t -i:8080)
```

### `uploads/` directory not being created

```bash
# Diagnosis — check write permissions:
ls -ld ~/storage-app/

# Fix:
chmod 755 ~/storage-app/
```

### Files not appearing after upload

```bash
# Verify multer received the field name 'file':
# The curl command MUST use -F "file=@filename"
# Wrong:  -F "image=@test.jpg"   ← field name mismatch
# Right:  -F "file=@test.jpg"
```
